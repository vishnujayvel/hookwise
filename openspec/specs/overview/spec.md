## Purpose

Hookwise is a config-driven middleware layer that intercepts Claude Code lifecycle events via
stdin/stdout JSON protocol, enforcing guard policies, recording analytics, and coaching developer
behavior in real time. It runs a background daemon to poll live data feeds (weather, news, calendar,
project context, insights) and surfaces aggregated metrics through a Python Textual TUI dashboard.
The system exists to give developers observability and behavioral guardrails over their AI-assisted
coding sessions without disrupting Claude Code's hot path.

## Requirements

### Requirement: SystemCapabilities

The system SHALL provide the following capabilities:
- Fail-open (ARCH-1): dispatch always exits 0 and never propagates errors to stdout — prevents hookwise failures from blocking Claude Code tool calls
- Single-writer SQLite with WAL mode and SetMaxOpenConns(1) (ARCH-2): deliberate choice to avoid concurrent-writer corruption; inherited from prior Dolt constraint but retained as correct under WAL
- Daemon writes JSON cache only, never the analytics DB (ARCH-3): decouples the feed polling process from the analytics write path, eliminating filesystem lock contention
- First-match-wins guard evaluation (ARCH-5): deterministic, predictable rule ordering with no combinatorial guard interaction
- FeedEnvelope canonical boundary type: centralizes Go→Python field naming to prevent silent schema drift (Bug #29 class: field name mismatches pass all single-language tests but break the TUI silently)
- Three-phase dispatch pipeline (declarative guards → handler guard/context → async side effects): allows fast-path guard decisions before expensive handler execution, and ensures side effects never delay the response returned to Claude Code
- Contract testing with byte-identical stdout JSON fixtures: enforces that the Claude Code integration protocol never silently regresses across refactors
- Pure-Go SQLite driver (modernc.org/sqlite): eliminates CGO dependency, simplifying cross-platform builds and removing the need for C toolchain in CI
- Dagger for CI: containerized pipeline is identical locally and in GitHub Actions, eliminating 'works on my machine' CI failures
- Cross-language boundary via atomic file write (not shared memory or socket): Python TUI reads a single flat JSON file; Go daemon writes it atomically — crash-safe and language-agnostic

## Technical Notes

- **Architecture Style**: Layered architecture: CLI entry points (cmd/hookwise) → dispatch/core engine (internal/core) → domain service layer (analytics, feeds, notifications, coaching) → SQLite persistence. The layering is justified by the need to keep the hot-path dispatch pipeline (called synchronously by Claude Code on every hook event) completely isolated from slower concerns like feed polling, TUI rendering, and analytics aggregation. A strict cross-language boundary separates Go (dispatch, daemon, analytics) from Python (TUI) via a JSON file cache rather than a shared process or socket.
- **Security Model**: No traditional authentication or authorization. The guard engine acts as the policy enforcement layer: declarative GuardRuleConfig rules intercept tool calls before execution, and BlockDangerousCommandsHandler matches Bash invocations against configurable dangerous-pattern lists. SecretScanningService detects accidental credential exposure in Write/Edit tool payloads by matching file paths against sensitive-file patterns and content against known API key regexes, emitting a warn GuardResult. Google Calendar integration uses OAuth2 with tokens persisted to disk; all other external APIs are unauthenticated public endpoints. Config files are local YAML with no network fetch; the set-secret script ensures secret values never enter conversation context or shell history.
- **External Integrations**: Claude Code (hook events via stdin/stdout JSON protocol; settings.json wiring), Open-Meteo API (weather feed — free, no API key required), Hacker News API (news feed — public, concurrent story fetch), Google Calendar API (OAuth2, token refresh, persisted in Python-compatible format), Anthropic Claude Haiku API (LLM-generated daily insights narrative, cached to disk), SQLite via modernc.org/sqlite (pure-Go, CGO-free, WAL mode — analytics, coaching, cost, notifications), Unix domain sockets (daemon IPC for health probe and graceful shutdown), Git CLI (project feed shells out to git for branch/commit/dirty-state context)
