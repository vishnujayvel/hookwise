# Analytics Specification

> Generated by openlore v1.0.0 on 2026-06-15 15:11
> Source files: internal/core/types.go, internal/analytics/analytics.go, internal/analytics/state.go, internal/analytics/sqlite.go

## Purpose

Tracks developer sessions, tool events, and AI authorship attribution; provides daily aggregations
and tool-usage breakdowns backed by a SQLite database.

## Entities

### AnalyticsEvent

> `internal/core/types.go`

A single recorded event in the analytics database. Captures tool usage, file edits, and AI
authorship signals for a session. *(analyzed in 2 chunks)*

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| sessionId | string | The Claude Code session this event belongs to |
| eventType | string | The hook event type that triggered this record |
| toolName | string | Name of the tool used (e.g. Edit, Bash) |
| timestamp | string | ISO 8601 timestamp of when the event occurred |
| filePath | string | Path to the file affected, if applicable |
| linesAdded | int | Number of lines added in a file edit event |
| linesRemoved | int | Number of lines removed in a file edit event |
| aiConfidenceScore | *float64 | 0.0–1.0 confidence that this change was AI-authored |

**Relationships:**

- aggregates-into SessionSummary (Multiple AnalyticsEvents for a session are summarized into a SessionSummary)
- references AIConfidenceScore (The aiConfidenceScore maps to the AIConfidenceScore classification thresholds)

### Session

> `internal/analytics/analytics.go`

Represents a bounded Claude Code coding session, tracking lifecycle timestamps and aggregated
metrics for tool usage, file edits, and AI authorship.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| id | string | Unique session identifier |
| started_at | time.Time | UTC timestamp when the session began |
| ended_at | time.Time | UTC timestamp when the session ended; absent for in-progress sessions |
| total_tool_calls | int | Total number of tool invocations during the session |
| file_edits_count | int | Number of file edits made during the session |
| ai_authored_lines | int | Lines attributed to AI authorship across all edits |
| human_verified_lines | int | Lines verified or written by the human developer |
| estimated_cost_usd | float64 | Estimated USD cost of the session's AI token usage |

**Relationships:**

- has-many Event (A session produces many tool invocation events)
- has-many AuthorshipEntry (A session accumulates many authorship ledger entries)

### Event

> `internal/analytics/analytics.go`

Records a single observable tool invocation or hook trigger within a session, capturing file impact
and AI confidence at point-in-time.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| session_id | string | Foreign key to the owning session |
| event_type | string | Category of hook event (e.g. PreToolUse, PostToolUse) |
| tool_name | string | Name of the Claude Code tool that was invoked; empty for non-tool events |
| timestamp | time.Time | UTC moment the event occurred |
| file_path | string | Path of the file affected, if any |
| lines_added | int | Lines inserted into the file by this event |
| lines_removed | int | Lines deleted from the file by this event |
| ai_confidence_score | float64 | 0.0–1.0 score indicating how confident the system is that this change was AI-authored |

**Relationships:**

- belongs to Session (Every event is scoped to exactly one session)

### AuthorshipEntry

> `internal/analytics/analytics.go`

Ledger record that attributes a set of file line changes to a specific tool within a session,
classifying the degree of AI versus human authorship.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| session_id | string | Foreign key to the owning session |
| file_path | string | Path of the file whose lines are being attributed |
| tool_name | string | Claude Code tool responsible for the change |
| lines_changed | int | Number of lines modified in this authorship record |
| ai_score | float64 | 0.0–1.0 score representing degree of AI authorship |
| classification | string | Derived label: high_probability_ai | likely_ai | mixed_verified | human_authored |
| timestamp | time.Time | UTC moment the change was recorded |

**Relationships:**

- belongs to Session (Each authorship entry is scoped to a session)

### CoachingState

> `internal/analytics/state.go`

Singleton record tracking the developer's active coaching session state: current mode, alert level,
tooling time accumulation, prompt history, and practice count. Persisted to a single SQLite row
(id=1) and scoped to a calendar day.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| LastPromptAt | *time.Time | Timestamp of the most recent coaching prompt delivered to the developer |
| PromptHistory | []string | Ordered list of past coaching prompts shown; stored as a JSON array |
| CurrentMode | string | Active developer mode (e.g. 'coding'). Defaults to 'coding' on first read |
| ModeStartedAt | *time.Time | Timestamp when the current mode was entered |
| ToolingMinutes | float64 | Accumulated minutes spent in tooling or non-coding activity today |
| AlertLevel | string | Current coaching alert severity ('none', 'yellow', 'orange', 'red'). Defaults to 'none' |
| TodayDate | string | ISO date string (YYYY-MM-DD) scoping this state to a calendar day |
| PracticeCount | int | Number of deliberate practice events completed today. Defaults to 0 |
| LastLargeChange | map[string]interface{} | Metadata about the most recent large code change detected; used to trigger coaching interventions. Stored as a JSON object |

### CostState

> `internal/analytics/state.go`

Singleton record tracking cumulative API token cost for the current calendar day. Automatically
resets the daily total when the stored date no longer matches today, ensuring the counter is always
scoped to a single day.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| TotalToday | float64 | Accumulated API cost in USD for TodayDate. Automatically reset to 0 when the date rolls over |
| TodayDate | string | ISO date string (YYYY-MM-DD) identifying the day this cost total applies to |

### Notification

> `internal/analytics/sqlite.go`

A user-facing alert generated by a named producer (budget, guard, coaching) with a TTL for automatic
expiry, surfaced in the TUI.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| id | string | Unique notification identifier |
| producer | string | Name of the producer that created the notification (budget, guard, coaching) |
| notification_type | string | Notification category (budget_threshold, guard_effectiveness, coaching_prompt) |
| content | string | Human-readable notification message |
| created_at | string | ISO 8601 creation timestamp |
| surfaced_at | string | ISO 8601 timestamp when the notification was shown to the user; null until surfaced |
| acted_on | boolean | Whether the user took action on this notification |
| branch | string | Git branch context when the notification was created |
| ttl_seconds | integer | Time-to-live in seconds; defaults to 86400 (24 hours) |
| expired | boolean | Computed: true when current time exceeds created_at plus ttl_seconds |

### AgentSpan

> `internal/analytics/sqlite.go`

An execution span for a subagent invoked within a Claude Code session, enabling tracing of nested
agent activity and file modifications.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| id | string | Unique span identifier |
| session_id | string | Foreign key referencing the parent session |
| agent_id | string | Identifier of the subagent instance |
| parent_agent_id | string | Agent ID of the invoking agent; null for root-level agents |
| agent_type | string | Type or role of the agent (e.g. Explore, general-purpose) |
| started_at | string | ISO 8601 timestamp when the span began |
| stopped_at | string | ISO 8601 timestamp when the span ended; null if still running |
| files_modified | string | JSON array of file paths modified by the agent during its span |

**Relationships:**

- belongs to Session (Span is scoped to a parent session)
- belongs to AgentSpan (Span may have a parent agent span for nested agent hierarchies)

## Requirements

### Requirement: AnalyticsEventValidation

The system SHALL validate AnalyticsEvent according to these rules:
- sessionId must be non-empty
- eventType must be a valid hook event type
- aiConfidenceScore, when present, must be between 0.0 and 1.0

#### Scenario: FileEditRecordedWithAIScore
- **GIVEN** A PostToolUse event for the Edit tool
- **WHEN** The analytics handler processes the event
- **THEN** An AnalyticsEvent is persisted with linesAdded, linesRemoved, and an aiConfidenceScore

### Requirement: SessionValidation

The system SHALL validate Session according to these rules:
- ended_at must be after started_at when present
- estimated_cost_usd must be non-negative
- All timestamps stored in UTC RFC3339 format

#### Scenario: StartASession
- **GIVEN** A unique session ID and a start timestamp
- **WHEN** StartSession() is called
- **THEN** A session record is inserted with started_at set; ended_at and stats fields are absent

#### Scenario: EndASession
- **GIVEN** An existing in-progress session
- **WHEN** EndSession() is called with an end timestamp and SessionStats
- **THEN** The session record is updated with ended_at and all aggregated stats (tool calls, edits, lines,
cost)

### Requirement: EventValidation

The system SHALL validate Event according to these rules:
- ai_confidence_score must be in [0.0, 1.0]
- timestamp must be stored in UTC RFC3339 format
- session_id must reference an existing session

#### Scenario: RecordAToolUseEvent
- **GIVEN** An active session and a completed tool invocation
- **WHEN** RecordEvent() is called with the EventRecord
- **THEN** An event row is persisted with tool_name, file_path, line deltas, and ai_confidence_score

#### Scenario: RecordANonToolEvent
- **GIVEN** An active session and a lifecycle hook with no tool name
- **WHEN** RecordEvent() is called with tool_name left empty
- **THEN** The event is persisted; tool_name is stored as empty string and excluded from ToolBreakdown queries

### Requirement: AuthorshipEntryValidation

The system SHALL validate AuthorshipEntry according to these rules:
- ai_score must be in [0.0, 1.0]
- classification is always derived from ai_score: >=0.8 → high_probability_ai, >=0.5 → likely_ai, >=0.2 → mixed_verified, <0.2 → human_authored
- classification is computed at write time and stored; it is not recomputed on read

#### Scenario: ClassifyAIDominantChange
- **GIVEN** An authorship entry with ai_score = 0.9
- **WHEN** RecordAuthorship() is called
- **THEN** The entry is persisted with classification 'high_probability_ai'

#### Scenario: ClassifyMixedChange

### Requirement: CoachingStateValidation

The system SHALL validate CoachingState according to these rules:
- Exactly one row with id=1 exists; REPLACE semantics enforce singleton invariant
- CurrentMode defaults to 'coding' when no row exists
- AlertLevel defaults to 'none' when no row exists
- TodayDate defaults to the current date (YYYY-MM-DD) when no row exists
- PromptHistory is serialized as a JSON array; malformed JSON is logged and discarded
- LastLargeChange is serialized as a JSON object; malformed JSON is logged and discarded
- Timestamp fields (LastPromptAt, ModeStartedAt) are stored as RFC3339 strings and parsed with ParseTimeFlex

#### Scenario: FirstReadReturnsDefaultState

### Requirement: Snapshot

The system SHALL creates a point-in-time copy of the live sqlite analytics database using vacuum into. resolves a collision-safe timestamped filename, creates the target directory if absent, and returns the absolute path of the written snapshot file.

> Implementation: `test_tab_snapshot` in `tui/tests/test_snapshots.py` · confidence: heuristic

#### Scenario: DirectoryDoesNotExist
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** The snapshots directory does not exist
- **WHEN** Snapshot is called
- **THEN** The directory is created with mode 0700 and the snapshot file is written successfully, returning its
absolute path

#### Scenario: EmptySnapshotsDirUsesDefault
- **GIVEN** snapshotsDir is an empty string
- **WHEN** Snapshot is called
- **THEN** DefaultSnapshotsDir() is used as the target directory

#### Scenario: FilenameCollisionAtSameTimestamp
- **GIVEN** A snapshot file with the current UTC timestamp already exists in the directory
- **WHEN** Snapshot is called

#### Scenario: CollisionSuffixCeilingExceeded

### Requirement: PruneSnapshots

The system SHALL enforces a retention policy by deleting the oldest snapshot files, keeping only the n most recent. returns the list of deleted paths. is a no-op when keep <= 0 or the number of snapshots is within the retention limit.

#### Scenario: KeepValueIsZeroOrNegative
- **GIVEN** keep is 0 or a negative integer
- **WHEN** PruneSnapshots is called
- **THEN** No files are removed and nil, nil is returned immediately

#### Scenario: SnapshotCountWithinRetentionLimit
- **GIVEN** The number of existing snapshots is less than or equal to keep
- **WHEN** PruneSnapshots is called
- **THEN** No files are removed and nil, nil is returned

#### Scenario: SnapshotCountExceedsRetentionLimit
- **GIVEN** There are more snapshots than keep allows
- **WHEN** PruneSnapshots is called
- **THEN** The oldest (len(snaps) - keep) files are deleted and their paths returned

#### Scenario: DeletionFailureMidPrune
<!-- openlore-test: tags=smoke,regression (auto) -->
- **GIVEN** One of the snapshot files cannot be removed (e.g. permissions error)
- **WHEN** PruneSnapshots is called
- **THEN** A wrapped error is returned along with the partial list of successfully pruned files

#### Scenario: NoSnapshotsDirectory
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** The snapshots directory does not exist
- **WHEN** PruneSnapshots is called with keep > 0
- **THEN** ListSnapshots returns empty; no error is returned and no files are removed

### Requirement: DefaultSnapshotsDir

The system SHALL returns the canonical default directory for snapshot files (~/.hookwise/snapshots), mirroring the convention used by defaultdbpath.

#### Scenario: ReturnsHomeRelativeAbsolutePath

- **WHEN** DefaultSnapshotsDir is called
