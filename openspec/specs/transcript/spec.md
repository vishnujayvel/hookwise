# Transcript Specification

> Generated by openlore v1.0.0 on 2026-06-15 15:11

## Purpose

Parses Claude Code session transcript files (.jsonl) and aggregates per-model token usage for cost
computation.

## Requirements

> `internal/transcript/transcript.go`

### Requirement: SumUsage

The system SHALL reads a jsonl transcript file line-by-line, filters assistant messages with usage data, and accumulates input/output/cache token counts keyed by model name.

#### Scenario: ValidTranscriptWithMultipleModels
<!-- openlore-test: priority=high tags=smoke (auto) -->
- **GIVEN** A JSONL transcript file containing assistant messages from two distinct models each with usage
fields
- **WHEN** SumUsage is called with the file path
- **THEN** Returns a map with one entry per model, each accumulating the sum of input, output, cache-read, and
cache-creation tokens across all assistant lines for that model

#### Scenario: NonAssistantLinesAreIgnored
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** A transcript file containing lines of type 'user', 'tool_result', or other non-assistant types
- **WHEN** SumUsage is called
- **THEN** Those lines are skipped and do not contribute to any model's token counts

#### Scenario: AssistantMessageWithMissingUsageField
<!-- openlore-test: priority=high tags=regression (auto) -->
- **GIVEN** An assistant-typed transcript line where the usage object is absent (null)
- **WHEN** SumUsage is called
- **THEN** That line is skipped without error and token counts are unaffected

#### Scenario: AssistantMessageWithMissingModelField
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** An assistant-typed transcript line where the model string is empty
- **WHEN** SumUsage is called
- **THEN** That line is skipped without contributing to any model bucket

#### Scenario: MalformedJSONLines
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** A transcript file containing one or more lines that are not valid JSON
- **WHEN** SumUsage is called
- **THEN** Malformed lines are silently skipped and valid lines are still processed

#### Scenario: EmptyTranscriptFile
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A transcript file that is empty or contains only blank lines
- **WHEN** SumUsage is called
- **THEN** Returns an empty (non-nil) map and a nil error

#### Scenario: FileDoesNotExist
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A path pointing to a non-existent file
- **WHEN** SumUsage is called
- **THEN** Returns nil map and a non-nil OS error

#### Scenario: ScannerReadErrorMidFile
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A valid file path but the underlying scanner encounters an I/O error partway through scanning
- **WHEN** SumUsage is called
- **THEN** Returns nil map and the scanner error

#### Scenario: LargeTranscriptFile
- **GIVEN** A transcript file with individual lines up to 10 MiB
- **WHEN** SumUsage is called
- **THEN** All lines are read correctly without truncation using the 10 MiB scanner buffer

#### Scenario: CacheTokenAccumulation
<!-- openlore-test: priority=high tags=smoke (auto) -->
- **GIVEN** Multiple assistant messages from the same model with non-zero cache_read_input_tokens and
cache_creation_input_tokens
- **WHEN** SumUsage is called
- **THEN** Cache read and cache creation tokens are summed separately into CacheReadTokens and
CacheCreationTokens in the returned Usage struct


## Sub-components

> `TranscriptReader` is an orchestrator. Each sub-component below implements one logical block.

### Sub-component: FileOpen

> Implements: `os.Open`

Opens the transcript .jsonl file for reading and returns a file handle for subsequent scanning.

#### Requirement: OpenTranscriptFile

The system SHALL open the file at the given path and return a readable file descriptor; must return an error if the path does not exist or is not readable.

#### Scenario: ValidPathOpensSuccessfully
<!-- openlore-test: tags=smoke,regression (auto) -->
- **GIVEN** a .jsonl file exists at the specified path with read permissions
- **WHEN** os.Open is called with that path
- **THEN** a non-nil *os.File is returned and error is nil

#### Scenario: MissingFileReturnsError
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** no file exists at the specified path
- **WHEN** os.Open is called with that path
- **THEN** a nil *os.File is returned and error is non-nil (os.ErrNotExist)

#### Scenario: UnreadableFileReturnsError
<!-- openlore-test: priority=high tags=regression (auto) -->
- **GIVEN** a file exists at the path but has no read permission (mode 000)
- **WHEN** os.Open is called with that path
- **THEN** a nil *os.File is returned and error is non-nil (permission denied)

### Sub-component: FileClose

> Implements: `f.Close`

Releases the file descriptor after transcript reading is complete to prevent resource leaks.

#### Requirement: ReleaseFileDescriptor

The system SHALL close the open file handle exactly once after all scanning operations complete; should be deferred immediately after a successful os.open to guarantee execution.

#### Scenario: ClosesOpenFileSuccessfully
<!-- openlore-test: tags=smoke,regression (auto) -->
- **GIVEN** a file handle returned by os.Open that has not been closed
- **WHEN** f.Close is called
- **THEN** the file descriptor is released and error is nil

#### Scenario: DeferredCloseExecutesAfterScanning
- **GIVEN** f.Close is registered via defer immediately after os.Open
- **WHEN** the enclosing function returns (normally or via error path)
- **THEN** f.Close is called exactly once regardless of the return path

### Sub-component: ScannerCreate

> Implements: `bufio.NewScanner`

Wraps the file reader in a buffered line scanner suitable for iterating over JSONL records.

#### Requirement: CreateLineScanner

The system SHALL create a bufio.scanner from the open file handle that splits input on newline boundaries; must return a non-nil scanner ready for sc.scan calls.

#### Scenario: ReturnsScannerForValidFile
- **GIVEN** an open, readable *os.File
- **WHEN** bufio.NewScanner is called with that file
- **THEN** a non-nil *bufio.Scanner is returned with default line-splitting behavior

#### Scenario: EmptyFileProducesValidScanner
- **GIVEN** an open file with zero bytes
- **WHEN** bufio.NewScanner is called with that file
- **THEN** a non-nil *bufio.Scanner is returned and the first sc.Scan call returns false

### Sub-component: BufferResize

> Implements: `sc.Buffer`

Overrides the scanner's default buffer capacity to accommodate long JSONL lines that exceed the 64 KB default.

#### Requirement: SetMaxTokenSize

The system SHALL set a buffer large enough to hold the longest expected jsonl line; should use a starting buffer of at least 64 kb and a max size of at least 1 mb to prevent bufio.errtoolong on large assistant messages.

#### Scenario: LargeLineScansWithoutError
<!-- openlore-test: tags=smoke,regression (auto) -->
- **GIVEN** a JSONL file containing a line of 500 KB
- **WHEN** sc.Buffer is called with max=1MB before scanning begins
- **THEN** sc.Scan successfully returns true and sc.Bytes returns the full line

#### Scenario: LineExceedingMaxReturnsError
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** a JSONL file containing a line larger than the configured max
- **WHEN** sc.Scan is called
- **THEN** sc.Scan returns false and sc.Err returns bufio.ErrTooLong

### Sub-component: ScanLine

> Implements: `sc.Scan`

Advances the scanner to the next line, returning true while lines remain and false on EOF or error.

#### Requirement: AdvanceToNextLine

The system SHALL return true and position the scanner on the next available line; must return false when eof is reached or an i/o error occurs; must not skip blank lines silently (blank lines should be detected via len check).

#### Scenario: ReturnsTrueForEachLine
- **GIVEN** a scanner over a file with N non-empty lines
- **WHEN** sc.Scan is called N times
- **THEN** each call returns true and sc.Bytes returns the corresponding line bytes

#### Scenario: ReturnsFalseAtEOF
- **GIVEN** a scanner that has consumed all lines
- **WHEN** sc.Scan is called one more time
- **THEN** sc.Scan returns false and sc.Err returns nil

#### Scenario: ReturnsFalseOnIOError
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** an underlying reader that returns an I/O error mid-file
- **WHEN** sc.Scan is called after the error occurs
- **THEN** sc.Scan returns false and sc.Err returns a non-nil error

### Sub-component: LineBytes

> Implements: `sc.Bytes`

Retrieves the raw bytes of the current scanned line for JSON unmarshalling.

#### Requirement: GetCurrentLineBytes

The system SHALL return the bytes of the most recently scanned line without a trailing newline; must only be called after sc.scan returns true; the returned slice must not be retained across subsequent sc.scan calls without copying.

#### Scenario: ReturnsLineWithoutNewline
- **GIVEN** a line 'foo bar\n' in the file
- **WHEN** sc.Bytes is called after sc.Scan returns true
- **THEN** the returned bytes equal []byte("foo bar") with no trailing newline

#### Scenario: SliceInvalidatedAfterNextScan
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** sc.Bytes was called and the result stored without copying
- **WHEN** sc.Scan is called again
- **THEN** the previously stored slice may reflect the new line content (undefined behavior unless copied)

### Sub-component: LineLength

> Implements: `len`

Guards against unmarshalling empty or whitespace-only lines that carry no token usage data.

#### Requirement: SkipEmptyLines

The system SHALL check the byte length of each scanned line before unmarshalling; should skip lines where len(line) == 0 to avoid passing empty input to json.unmarshal.

#### Scenario: ZeroLengthLineIsSkipped
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** a JSONL file with a blank line between two valid JSON objects
- **WHEN** len is evaluated on the blank line's bytes
- **THEN** len returns 0 and the line is skipped without calling json.Unmarshal

#### Scenario: NonEmptyLineProceeds
- **GIVEN** a line containing a valid JSON object
- **WHEN** len is evaluated on the line's bytes
- **THEN** len returns a positive integer and json.Unmarshal is called with the bytes

### Sub-component: JsonParse

> Implements: `json.Unmarshal`

Decodes each JSONL line into a structured record to extract per-model token usage fields.

#### Requirement: DeserializeTokenRecord

The system SHALL unmarshal the raw json bytes into a target struct containing model identifier and token usage fields; must return an error on malformed json; should silently skip records where token usage fields are absent (zero-value structs are valid).

#### Scenario: ValidRecordPopulatesStruct
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** a JSON line with model, inputTokens, and outputTokens fields
- **WHEN** json.Unmarshal is called
- **THEN** the target struct fields are populated correctly and error is nil

#### Scenario: MalformedJSONReturnsError
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** a line containing invalid JSON such as '{unclosed'
- **WHEN** json.Unmarshal is called
- **THEN** a non-nil error is returned and the target struct is unchanged

#### Scenario: MissingTokenFieldsProduceZeroValues
<!-- openlore-test: priority=high tags=regression (auto) -->
- **GIVEN** a valid JSON line that omits inputTokens and outputTokens
- **WHEN** json.Unmarshal is called
- **THEN** error is nil and the token fields on the struct are 0

#### Scenario: UnknownFieldsAreIgnored
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** a JSON line with extra fields not present in the target struct
- **WHEN** json.Unmarshal is called
- **THEN** error is nil and only known fields are populated

### Sub-component: ScanError

> Implements: `sc.Err`

Checks for any I/O or buffer-overflow error that terminated the scan loop before EOF.

#### Requirement: DetectScanFailure

The system SHALL be called after the scan loop exits; shall return nil if scanning ended due to normal eof; must return a non-nil error if scanning was interrupted by an i/o error or a line exceeding the buffer maximum; callers must propagate this error rather than treating partial reads as complete.

#### Scenario: NilErrorOnCleanEOF
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** a well-formed JSONL file that was fully read without errors
- **WHEN** sc.Err is called after the scan loop
- **THEN** sc.Err returns nil

#### Scenario: NonNilErrorOnIOFailure
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** an underlying reader that returned an I/O error before EOF
- **WHEN** sc.Err is called after the scan loop
- **THEN** sc.Err returns a non-nil error describing the failure

#### Scenario: BufferOverflowErrorPropagates
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** a file line that exceeded the configured buffer maximum
- **WHEN** sc.Err is called after the scan loop
- **THEN** sc.Err returns bufio.ErrTooLong

#### Scenario: PartialResultsDiscardedOnError
<!-- openlore-test: priority=high tags=regression (auto) -->
- **GIVEN** a file where the first 10 lines parsed successfully but line 11 caused an I/O error
- **WHEN** sc.Err is called and returns non-nil
- **THEN** the caller propagates the error and does not return partial token aggregates as a complete result
