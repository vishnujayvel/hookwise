# Pricing Specification

> Generated by openlore v1.0.0 on 2026-06-15 15:11
> Source files: internal/pricing/pricing.go

## Purpose

Computes estimated USD cost from Anthropic token usage for a given model ID, mapping model IDs to
rate families (opus/sonnet/haiku) and applying optional per-family rate overrides.

## Entities

### Usage

> `internal/pricing/pricing.go`

Token consumption record for a single Anthropic model invocation, tracking the four billing buckets
used to compute USD cost.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| InputTokens | int64 | Number of input tokens sent to the model |
| OutputTokens | int64 | Number of output tokens generated by the model |
| CacheReadTokens | int64 | Number of tokens read from the prompt cache, billed at a reduced rate |
| CacheCreationTokens | int64 | Number of tokens written to the prompt cache, billed at a premium rate |

**Relationships:**

- used-by ModelPricing (Usage token counts are multiplied against ModelPricing rates to produce a USD cost)

### ModelPricing

> `internal/pricing/pricing.go`

USD pricing rates per million tokens for a single Claude model family, covering four billing
dimensions: input, output, cache-read, and cache-write.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| Input | float64 | USD per million input tokens (Opus: $15.00, Sonnet: $3.00, Haiku: $0.80) |
| Output | float64 | USD per million output tokens (Opus: $75.00, Sonnet: $15.00, Haiku: $4.00) |
| CacheRead | float64 | USD per million cache-read tokens (Opus: $1.50, Sonnet: $0.30, Haiku: $0.08) |
| CacheWrite | float64 | USD per million cache-write tokens (Opus: $18.75, Sonnet: $3.75, Haiku: $1.00) |

**Relationships:**

- belongs to ModelFamily (Each ModelPricing table is associated with exactly one model family)
- applied-to Usage (Rates are multiplied against Usage token counts to compute total USD cost)

### ModelFamily

> `internal/pricing/pricing.go`

Classification of a Claude model into one of three pricing tiers—Opus, Sonnet, or Haiku—determined
by case-insensitive substring matching on the model ID, with Sonnet as the fallback.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| name | string | Family identifier: one of 'opus', 'sonnet', or 'haiku' |

**Relationships:**

- has ModelPricing (Each ModelFamily maps to a built-in ModelPricing rate table)

## Requirements

### Requirement: UsageValidation

The system SHALL validate Usage according to these rules:
- All token counts must be non-negative integers
- Zero usage always produces a cost of exactly 0.0

#### Scenario: ComputeCostForRecognizedModel
- **GIVEN** A Usage record with non-zero token counts and a model ID containing a recognized family substring
- **WHEN** Compute() is called
- **THEN** Returns the USD cost calculated from per-MTok rates matched to the resolved model family

#### Scenario: ComputeCostForUnrecognizedModel
- **GIVEN** A Usage record and a model ID that does not contain 'opus', 'sonnet', or 'haiku'
- **WHEN** Compute() is called
- **THEN** Falls back to Sonnet rates and returns a non-zero USD cost without panicking

#### Scenario: ZeroUsageYieldsZeroCost
- **GIVEN** A Usage record with all four token counts set to zero
- **WHEN** Compute() is called with any model ID
- **THEN** Returns exactly 0.0

### Requirement: ModelPricingValidation

The system SHALL validate ModelPricing according to these rules:
- Override keys must follow the pattern '<family>.<rate-type>' where family is one of 'opus', 'sonnet', 'haiku' and rate-type is one of 'input', 'output', 'cache_read', 'cache_write'
- Unknown override keys are silently ignored; they do not cause errors
- Only keys explicitly present in the overrides map replace built-in values; absent keys retain their defaults
- A nil or empty overrides map is a no-op and produces results identical to built-in rates

#### Scenario: ApplySelectiveRateOverride
- **GIVEN** Built-in Sonnet rates and an overrides map containing 'sonnet.output' set to a custom value
- **WHEN** ComputeWithRates() is called with a Sonnet model ID
- **THEN** The output rate is replaced with the override value while all other Sonnet rates remain at their
built-in values

#### Scenario: EmptyOverridesIdenticalToBuiltInCompute
- **GIVEN** A nil or empty overrides map and a Usage record
- **WHEN** ComputeWithRates() is called
- **THEN** Produces results identical to calling Compute() with the same model and usage

#### Scenario: OverrideForADifferentFamilyIsIgnored
- **GIVEN** An overrides map containing 'haiku.input' and a Sonnet model ID
- **WHEN** ComputeWithRates() is called
- **THEN** The Sonnet input rate is unchanged; the haiku override is silently ignored

### Requirement: ModelFamilyValidation

The system SHALL validate ModelFamily according to these rules:
- Model IDs are matched in strict priority order: opus is checked before sonnet, sonnet before haiku
- Matching is case-insensitive; 'CLAUDE-OPUS-4-20250601' and 'claude-opus-4-8' both resolve to Opus
- A model ID that contains none of the three family substrings defaults to Sonnet
- If a model ID contains multiple family substrings, the highest-priority match wins (e.g., a string containing both 'opus' and 'sonnet' resolves to Opus)

#### Scenario: MatchOpusModelCaseInsensitively
- **GIVEN** A model ID containing the substring 'opus' in any casing
- **WHEN** the model ID is resolved to a family
- **THEN** The Opus family is selected and Opus rates apply ($15.00 input, $75.00 output, $1.50 cache-read,
$18.75 cache-write)

#### Scenario: FallbackToSonnetForUnrecognizedModel
- **GIVEN** A model ID that does not contain 'opus', 'sonnet', or 'haiku'
- **WHEN** the model ID is resolved to a family
- **THEN** The Sonnet family is selected as the default fallback without panicking

#### Scenario: PriorityOrderEnforcedOnAmbiguousID
- **GIVEN** A model ID whose lowercase form contains both 'opus' and 'sonnet'
- **WHEN** the model ID is resolved to a family
- **THEN** The Opus family is selected because 'opus' is evaluated before 'sonnet'

#### Scenario: MatchHaikuModel
- **GIVEN** A model ID containing the substring 'haiku' but not 'opus' or 'sonnet'
- **WHEN** the model ID is resolved to a family
- **THEN** The Haiku family is selected and Haiku rates apply ($0.80 input, $4.00 output, $0.08 cache-read,
$1.00 cache-write)

### Requirement: Compute

The system SHALL returns the estimated usd cost for a model and token usage using built-in rate tables. unknown model ids fall back to sonnet rates. zero usage shall return 0.0. shall never panic.

#### Scenario: OpusModelCost
- **GIVEN** A model ID containing 'opus' and non-zero token usage
- **WHEN** Compute is called
- **THEN** Returns cost computed at Opus rates (input $15/MTok, output $75/MTok, cache-read $1.50/MTok,
cache-write $18.75/MTok)

#### Scenario: SonnetModelCost
- **GIVEN** A model ID containing 'sonnet' and non-zero token usage
- **WHEN** Compute is called
- **THEN** Returns cost computed at Sonnet rates (input $3/MTok, output $15/MTok, cache-read $0.30/MTok,
cache-write $3.75/MTok)

#### Scenario: HaikuModelCost
- **GIVEN** A model ID containing 'haiku' and non-zero token usage
- **WHEN** Compute is called
- **THEN** Returns cost computed at Haiku rates (input $0.80/MTok, output $4/MTok, cache-read $0.08/MTok,
cache-write $1/MTok)

#### Scenario: UnknownModelFallsBackToSonnet
- **GIVEN** A model ID that matches no known family (e.g. 'claude-unknown-x')
- **WHEN** Compute is called
- **THEN** Returns cost computed at Sonnet rates without panicking

#### Scenario: ZeroUsage
- **GIVEN** Any model ID and a Usage with all token counts set to zero
- **WHEN** Compute is called
- **THEN** Returns exactly 0.0

#### Scenario: CaseInsensitiveModelMatching
- **GIVEN** A model ID in uppercase such as 'CLAUDE-OPUS-4-20250601'
- **WHEN** Compute is called
- **THEN** Matches the Opus family and returns cost at Opus rates

### Requirement: ComputeWithRates

The system SHALL returns the estimated usd cost, applying per-family rate overrides from a supplied map before computing. override keys use the form '<family>.<rate-type>'. unknown override keys are silently ignored. a nil or empty override map must produce identical results to compute. shall never panic.

#### Scenario: NilOverridesMatchesCompute
- **GIVEN** Any model ID, token usage, and a nil overrides map
- **WHEN** ComputeWithRates is called
- **THEN** Returns the same value as Compute called with the same model and usage

#### Scenario: OverrideSingleRate
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** A Sonnet model, non-zero usage, and an overrides map with key 'sonnet.input' set to 1.00
- **WHEN** ComputeWithRates is called
- **THEN** Input token cost is computed at $1.00/MTok while all other Sonnet rates remain at built-in values

#### Scenario: OverrideMultipleRatesForTheMatchedFamily
- **GIVEN** An Opus model and overrides map containing 'opus.output' and 'opus.cache_write'
- **WHEN** ComputeWithRates is called
- **THEN** Output and cache-write costs use the override values; input and cache-read use built-in Opus rates

#### Scenario: OverrideForNonMatchedFamilyIsIgnored
- **GIVEN** A Haiku model and an overrides map containing only 'opus.input'
- **WHEN** ComputeWithRates is called
- **THEN** Cost is computed at built-in Haiku rates; the opus override key is silently ignored

#### Scenario: UnknownOverrideKeyIsSilentlyIgnored
- **GIVEN** Any model and an overrides map with an unrecognised key such as 'sonnet.unknown'
- **WHEN** ComputeWithRates is called
- **THEN** Unrecognised key is ignored; cost is computed from remaining resolved rates without panicking

### Requirement: MatchFamily

The system SHALL resolves a model id string to one of three pricing families (opus, sonnet, haiku) via case-insensitive substring search in priority order. unrecognised model ids fall back to the default family (sonnet).

#### Scenario: OpusSubstringMatch
- **GIVEN** A model ID containing the substring 'opus' in any case
- **WHEN** matchFamily is called
- **THEN** Returns the opus family

#### Scenario: SonnetSubstringMatch
- **GIVEN** A model ID containing 'sonnet' but not 'opus'
- **WHEN** matchFamily is called
- **THEN** Returns the sonnet family

#### Scenario: HaikuSubstringMatch
- **GIVEN** A model ID containing 'haiku' but not 'opus' or 'sonnet'
- **WHEN** matchFamily is called
- **THEN** Returns the haiku family

#### Scenario: OpusTakesPriorityOverSonnet
- **GIVEN** A hypothetical model ID containing both 'opus' and 'sonnet'
- **WHEN** matchFamily is called
- **THEN** Returns the opus family (priority order: opus first)

#### Scenario: UnknownModelFallsBackToSonnet
- **GIVEN** A model ID that contains neither 'opus', 'sonnet', nor 'haiku'
- **WHEN** matchFamily is called
- **THEN** Returns the sonnet family as the default fallback
